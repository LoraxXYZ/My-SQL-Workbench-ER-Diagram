# 10. Implementace

## 1. Návrh schématu v MySQL Workbench

Databáze byla navržena v programu MySQL Workbench pomocí ER diagramu.

Byly vytvořeny tyto tabulky:

Team – týmy
Match – zápasy
Customer – zákazníci
Ticket – vstupenky
Vztahy:
Team → Match = 1:N
(jeden tým může hrát více zápasů)
Match → Ticket = 1:N
(na jeden zápas je více vstupenek)
Customer → Ticket = 1:N
(jeden zákazník může mít více vstupenek)
## 2. Lokální prostředí

Pro běh databáze byl použit např.:

XAMPP nebo WampServer
databázový server MySQL / MariaDB
rozhraní phpMyAdmin

Po spuštění serveru je databáze dostupná na:

http://localhost/phpmyadmin
## 3. Přenesení schématu do databáze
a) Generování SQL z Workbench
Database → Forward Engineer
Vygeneruje se SQL skript (CREATE TABLE ...)
Ten odpovídá tvému finálnímu kódu
b) Import do phpMyAdmin
Vytvoříš databázi (např. mydb)
Klikneš na Import
Nahraješ SQL soubor
Hotovo – tabulky se vytvoří
## 4. Výsledná struktura (TVŮJ model)
Team
sloupec	význam
id_team	ID týmu
name	název týmu
Match
sloupec	význam
id_match	ID zápasu
match_date	datum
home_team_id	domácí tým
away_team_id	hosté

vazba:

home_team_id → Team
away_team_id → Team
Customer
sloupec	význam
id_customer	ID zákazníka
name	jméno
email	email
Ticket
sloupec	význam
id_ticket	ID vstupenky
seat	místo
price	cena
id_customer	zákazník
id_match	zápas

vazby:

id_customer → Customer
id_match → Match
